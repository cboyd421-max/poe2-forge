How to use POE2 Forge
1 · Start the proxy
The Optimizer needs a local proxy running to talk to the GGG trade API. Open a terminal in the folder containing poe2forge-proxy.js and run:

node poe2forge-proxy.js

The proxy listens on http://localhost:3001. The status dot in the top-right corner of the header turns green when it's reachable.
2 · Configure your POESESSID
Forge uses your GGG account cookie (POESESSID) to make authenticated trade-API requests. To get it:

1. Log in to pathofexile.com in your browser
2. Open DevTools (F12) → Application → Cookies → pathofexile.com
3. Copy the value of the POESESSID cookie
4. Paste it into your .env file next to the proxy:
   POESESSID=your_cookie_value_here
5. Restart the proxy

The .env file should already be in your .gitignore — never commit it. Full GGG OAuth flow lands in Phase 1 when GGG approves the app.
3 · Import a character
Use the Connect Character section above. The League dropdown auto-fills from GGG when the proxy is reachable. Pick your character, click Import →, and the Optimizer tab fills with your live gear.

The shared currentBuild object updates on import — every other tab that reads from it (Optimizer paperdoll, Theorycraft scanners in later phases) sees the same source of truth. Click Logout to clear it.
4 · Troubleshooting
Proxy offline: Check the terminal where you ran node poe2forge-proxy.js for errors. Most common: Node version too old (need 18+), or port 3001 already in use.

League badge shows "PROXY OFFLINE": The proxy isn't reachable. Make sure it's running and that nothing else is using port 3001.

POESESSID expired: GGG cookies expire periodically. Re-do step 2 to refresh.

Nothing happens when I click a paperdoll slot: You need an imported build first. Use Welcome → Import (Phase 1) or Theorycraft → PoB Decoder (Phase 2).
